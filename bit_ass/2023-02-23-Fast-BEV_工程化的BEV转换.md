# Fast-BEV 工程化的BEV转换

 [*Link:*](https://zhuanlan.zhihu.com/p/608755921)

商汤最新出的一个3D转换模块，按照之前[https://zhuanlan.zhihu.com/p/604293845](https://zhuanlan.zhihu.com/p/604293845)属于静态转换。但是其利用了转换过程中增加冗余信息为动态任务提供了空间。这个思路简单粗暴但有效，一个非常好的R&D范例。

从我的角度来看，对于缺点：由于是通过冗余信息，BEV 后的网络需要负责支持动态任务，这就需要增加 BEV 网络的 capacity。但是，由于本身速度非常快，BEV 额外增加网络规模带来的时间损耗都可以赚回来，整体还是省时间。

论文自身提出的工作是3点：

1. 数据增强
2. 转换模块
3. 时序融合

指标提升来源整体的各项工作，但是不少来源都来自于转换模块带来的间接优势：

* Voxel / Image Resolution / 2D Backbone 由于转换模块省了不少时间，这些因素就可以增加，带来指标提升。
* 模块内无学习参数，让过拟合发生的更加延后。2D 和 BEV augmentation 即使区分开，由于中间的模块没参数，也不需要 aug。

对于转换的优化主要是两点：静态化带来的预运算、BEV 的直接融合。
